## 1. 全角・半角・句読点・スペース

**1-1. 日本語の句読点には全角の「。」と「、」を使う**

**1-2. すべての英数字、符号、疑問符、感嘆符は半角文字を使う**

| OK | NG |
|---|---|
| 準備をしましょう ! | 準備をしましょう!(記号は半角だが前にスペースがない) |
| 準備をしましょう ! | 準備をしましょう!(記号が全角) |

**1-3. 数字は、慣習的に使われている場合を除いて、半角の算用数字(1、2、3)を使う**

例: `バージョン5.5`、`二次元コード`(「二次元」は慣習的な漢数字表記なのでそのまま)

**1-4. 数字を除く半角文字と全角文字の間には、半角文字1字分のスペースを入れる**

ただし例外として『』「」。、の**前後**、コロン記号(:)の**前**には入れない。また、文字列最先頭のスペースも不要。

| OK | NG |
|---|---|
| WordPress の使い方 | WordPressの使い方(半角文字の後にスペースがない) |
| ユーザー ID: username | ユーザー ID : username(コロンの前にスペースは不要) |
| こんにちは、username さん。 | こんにちは、 username さん。(、の後のスペースは不要) |
| 準備ができましたか ? | 準備ができましたか?(半角記号の前にスペースがない) |

**1-5. 丸括弧 ( ) は半角にし、前後には半角文字1文字分のスペースを入れる**

例: `ユーザー username からのコメント (2010年5月10日12時30分)。`

括弧に関する詳細例は[コアの翻訳](https://translate.wordpress.org/projects/wp/dev/ja/default/?filters%5Bterm%5D=%28&filters%5Bterm_scope%5D=scope_any&filters%5Bstatus%5D=current_or_waiting_or_fuzzy_or_untranslated&filters%5Buser_login%5D=&filter=Apply+Filters&sort%5Bby%5D=original&sort%5Bhow%5D=desc)を参照。

**1-6. 丸括弧 ( ) の内側にはスペースを入れない**

**1-7. 丸括弧内には句点は付けず、複数の文がある場合には各文の間にだけ句点を付ける**

**1-8. 丸括弧が末尾にくる場合は、括弧の外に句点を付ける**

**1-9. 半角数字の前後には半角スペースを入れない**

| OK | NG |
|---|---|
| 1件のコメント。 | 1 件のコメント。(半角数字の後にスペースが入っている) |
| 33件のエラー。 | ３３件のエラー。(数字が全角になっている) |
| %d件の投稿、%d個の文字列 | %d 件の投稿、%d 個の文字列(半角数字で置き換えられることになるプレースホルダー後のスペースは不要) |

このルールが適用されるのは**数字、および数字に置き換わるプレースホルダー(`%d` `%1$d` など)**のみ。文字列に置き換わる `%s` の扱いは「6. プレースホルダー」のスペースルールを参照(こちらは 1-4 の「数字を除く半角文字」扱いで、前後にスペースを入れる)。

## 2. 括弧・カギ括弧・引用符

**2-1. 原文が大文字で始まる単語で固有名詞を区別している部分は「」で囲む**

原文でメニュー項目やボタンラベルを大文字始まりで区別している場合、日本語では大文字・小文字での差別化ができないため、カギ括弧を使う。

**2-2. メニュー項目やボタンラベルは「」で囲む**

**2-3. ""(二重引用符)で囲まれた単語の扱い**

ドメイン名や関数名など英字の場合はそのままの引用符を使い、日本語の場合は「」を使う。

例: `「ツール → 利用可能なページ」ページで "Press This" をインストールします。「Press This を開く」ボタンをクリックしてください。`

**2-4. `<em>` や `<i>` により日本語表示が斜体になってしまう場合**

「」などを使って強調性を表現する。場合によっては省略してもよい。

## 3. カタカナ語の長音記号

**3-1. 長音記号「ー」は、長音を含めて4文字以下になる単語には付ける**

5文字以上になる単語には付けない。

例: `ユーザー`、`サーバー`、`コンパイラ`、`インストーラ`、`コンテナ`

**3-2. 合成語の場合は、各要素に対してこの考え方を適用する**

例: `ユーザーインターフェース`

ただし、単語の末尾が er, ar, or, re, y, ew となるもののうち、日本語の末尾が拗音になるものや日本語で定着しているものには、例外として付ける。

例: `コミュニティー`、`アーキテクチャー`、`アップローダー`、`コンピューター`、`エディター`

## 4. 中点「・」

カタカナの複合語には、用語集などで特に指定しないかぎり、原則として中点「・」は使用しない。カタカナ語が長くなり過ぎて意味がわかりにくくなる場合は、「の」で区切るなどの工夫をする。

| OK | NG |
|---|---|
| マニュアルの章、節、および項の名前は、「」で示します。 | マニュアルの章・節・項の名前は、「」で示します。 |
| テキストエディタのベースウィンドウが表示されます。 | テキストエディタ・ベースウィンドウが表示されます。 |

例外として、カタカナが続いて区切りがわかりにくくなる場合は、中点を使ってもかまわない(例: `ライブラリ・リファレンスマニュアル`)。

## 5. 日付の翻訳

WordPressはPHPの[dateフォーマット](https://www.php.net/manual/ja/datetime.formats.date.php)を使って日付表示を翻訳できるようにしている。文字列にdateフォーマットが含まれる場合は、フォーマット自体を**コードとして保持**し翻訳しない。

例:
- `m/d/Y g:i:s A` → 表示結果は `01/15/2014 6:20:40 PM`
- `j F, Y @ G:i:s` → 表示結果は `15 January, 2014 @ 18:20:40`

一般的な日時の表記方法・空白の取り方:

- `2014年1月1日 (水)`
- `2014年1月1日2:00 AM`

## 6. プレースホルダー

`%s`(文字列)、`%d`(数値)、`%1$s` `%2$s`(順序付き)などは、訳文でも**数・種類を完全一致**させる。

チェック方法:
1. 原文からプレースホルダーを全て抜き出す
2. 訳文からプレースホルダーを全て抜き出す
3. 両者のリストが(順序は変わってもよいが種類と個数は)一致しているか確認する

語順を変える必要がある場合は、`%s %s` のような無番号プレースホルダーを `%1$s` `%2$s` の番号付きに変換してから並べ替える。

**6-1. プレースホルダー前後のスペースは「何に置き換わるか」で決まる**

置き換わる内容によって、1-4(半角文字はスペースあり)と 1-9(数字はスペースなし)のどちらが適用されるかが変わる。

| 種類 | 扱い | 例(いずれも公式スタイルガイド掲載) |
|---|---|---|
| `%d` `%1$d` など数値に置き換わるもの | 数字と同じ扱い。**前後にスペースを入れない**(1-9) | `%d件の投稿、%d個の文字列` |
| `%s` `%1$s` など文字列(名前・メールアドレス等の半角文字列)に置き換わるもの | 半角文字と同じ扱い。**前後に半角スペースを入れる**(1-4) | `認証メールを %s へ送信しました。`(表示例: `認証メールを you@example.com へ送信しました。`) |

よくある誤解への注意:

- 1-4 の例「こんにちは、username さん。」はリテラルの半角単語の例で、`%s` もこれと同じ扱いになる。「プレースホルダーだから一律スペースなし」**ではない**(スペースなしが明記されているのは数値系のみ)
- `%s` が日本語の文字列(投稿タイトルなど)に置き換わることが文脈上明らかな場合のスペースの有無は、公式スタイルガイドに明示の規定がない。同一プロジェクトの既存訳や [Consistency Tool](https://translate.wordpress.org/consistency) に合わせ、判断がつかなければ `[要確認]` とする
- `scripts/validate_po.py` の `NUM_SPACING` チェックは数値系(`%d` 等)と生の数字のみを対象にしており、`%s` 直後のスペースは違反として扱わない
